# 07 - Modelo de negocio

## El producto SaaS (foco principal)
Web de pedidos online + sistema de gestión a medida para hamburgueserías.

### Precio
| Concepto | Monto |
|---|---|
| Instalación (mes 1, único pago) | USD 150 |
| Suscripción mensual | USD 35 |

### Por qué este precio es razonable para el cliente
- Un combo de burger hoy sale mínimo $15.000 ARS
- USD 35/mes ≈ 3 burgers. Venden mínimo 30/día, picos de 150 (caso TCocina domingo)
- El sistema les devuelve tiempo, reduce errores y fideliza clientes → se paga solo

### Diferenciales del producto vs competencia
- **Fidelización** ("soles" / figuritas): casi nadie lo tiene
- **Seguimiento de pedido en tiempo real**: el cliente no pregunta más
- **Microturnos dinámicos**: control de capacidad por horario
- **Vista de cocina** dedicada
- **Reseñas de productos** con moderación
- **Panel de admin completo**: pedidos, cupones, turnos, productos, lealtad
- **Personalización real**: no es un template genérico

## El modelo de agencia (ingreso paralelo)
Webs a medida para cualquier negocio: gastronomía, distribuidoras, arquitectura, etc.
Proyectos puntuales, ingreso directo. Sirven para financiar mientras crece el SaaS.

## Meta
Conseguir clientes mensuales recurrentes del SaaS. Cada cliente = USD 35/mes pasivo.
10 clientes = USD 350/mes. 30 clientes = USD 1.050/mes. Y sigue escalando.

## Expansión: más allá de las hamburgerías

[[Doña Clara]] es el primer proyecto de agencia en gastronomía que **no es hamburguesería** y aun así usa la misma arquitectura core: pedidos + comandas + WhatsApp. Esto abre un nuevo segmento para el SaaS:

- Casas de comidas / rotiserías
- Restós simples con menú del día
- Emprendimientos de comida casera con delivery

> [!note]
> Si Doña Clara sale bien → es el argumento para ampliar el target del SaaS. No "para hamburgerías" sino "para cualquier cocina con volumen". El modelo de precio sigue siendo el mismo.
